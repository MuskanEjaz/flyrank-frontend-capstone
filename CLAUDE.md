# AI Development Rules

## Project Stack

- Use React with Vite and JavaScript.
- Use plain CSS unless another styling system is explicitly approved.
- Keep components focused and reusable.

## Implementation Rules

1. Inspect existing files before editing.
2. Write a short implementation plan before making significant changes.
3. Do not add unrelated features, packages, routes, or backend code.
4. Use semantic HTML and visible labels.
5. Associate validation errors with fields using accessible ARIA attributes.
6. Handle empty, whitespace-only, invalid, minimum-length, and maximum-length inputs.
7. Never accept AI-generated code without manual review.

## Verification Rules

- Run automated tests after implementation.
- Run the production build before committing.
- Fix test, runtime, lint, and build errors before considering work complete.
- Use Conventional Commits for all Git commits.
