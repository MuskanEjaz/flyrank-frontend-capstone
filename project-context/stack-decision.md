# Stack Decision

For the FlyRank portfolio, I chose Next.js with Vercel because it fits the constraints I am working within and gives me the strongest balance of clarity, polish, and maintainability.

The main constraints are simple: I need something free, I want to use React and Next.js because that is the skill level I am building toward, and I need to support the portfolio sections I have in mind, including images, demos, GitHub links, and case studies. I also do not need a backend yet, so I want a setup that stays lightweight and focused on frontend presentation.

I compared three options.

First, HTML/CSS/JS with GitHub Pages. This option is very lightweight and free, and it would be easy to publish quickly. However, it would be harder to structure a more polished portfolio experience over time. It would also be less convenient for reusable components, section-based page organization, and future growth as the portfolio becomes more detailed.

Second, React/Vite with Vercel. This option is strong because it uses React and gives me a modern component-based workflow. It would also be a good fit for a portfolio that needs strong visual presentation and interactive sections. I did not choose it because I want to work in a setup that is closer to the full React/Next.js direction I am learning, and Next.js gives me a more complete framework structure for routing, layout, and long-term maintainability.

Third, Next.js with Vercel. This is the option I chose because it aligns with my skill level, stays free to use for the portfolio scope, and gives me the best foundation for building a polished and maintainable site. It supports a clear structure for sections, reusable layouts, and thoughtful presentation without introducing backend complexity. It also gives me strong display quality for images, content sections, and demos while keeping the project organized.

The trade-off is that Next.js is a little heavier than plain HTML/CSS/JS, and it is more framework-oriented than a minimal static site. I see that as a worthwhile trade-off because it helps me build something that feels more professional and easier to extend later. I am also not adding backend features right now, so the extra structure does not create unnecessary complexity for the current scope.

In short, I selected Next.js with Vercel because it best matches my constraints, supports the portfolio experience I want to create, and gives me a solid foundation that is easier to maintain and improve over time without inventing achievements or overstating what is already available.
